# Постановка задачи

При разработке информационной системы необходимо реализовать клиентскую часть в виде Web-приложения для подачи заявок на ремонт оргтехники (в том числе без регистрации — форма быстрой заявки на главной странице) и административную часть системы для управления заявками на ремонт, клиентами, товарами, запчастями, поставщиками, продажами, сотрудниками и учётными данными, а также для формирования отчётности. Информационная система должна поддерживать ролевую модель пользователей, включающую следующие роли: администратор системы, менеджер, инженер, бухгалтер, директор, клиент. Информационная система должна быть реализована с использованием реляционной SQL-базы данных и программного интерфейса (API) для обеспечения взаимодействия клиентских приложений с базой данных.
