# Методы

При разработке информационной системы «ВузяПринт» применяются современные методы и технологии создания программного обеспечения, направленные на повышение производительности, надёжности и удобства использования системы. Реализация опирается на реляционную модель данных, трёхуровневую архитектуру приложения и единый подход к доступу к данным через слой ORM.

База данных информационной системы спроектирована в соответствии с принципами реляционного моделирования и нормализована до третьей нормальной формы (3НФ). Между сущностями реализованы связи «один ко многим» и при необходимости «многие ко многим» (через промежуточные таблицы); целостность и согласованность данных обеспечиваются внешними ключами и правилами каскадного и ограниченного удаления. Хранение информации организовано с помощью сущностей, описывающих предметную область: заявки на ремонт (заказы) для приёма, ведения и закрытия заявок; клиенты как контрагенты организации; товары и запчасти как номенклатура с привязкой к поставщикам и остаткам; поставщики как справочник поставщиков товаров и запчастей; сотрудники и должности для кадрового учёта; виды работ как справочник услуг (диагностика, ремонт, заправка и т.д.); работы по заявкам и позиции заявок по запчастям как состав заявки; продажи и позиции продаж для оформления продаж и состава продажи; пользователи и роли для учётных записей и разграничения доступа (ASP.NET Core Identity). Такой подход обеспечивает централизованное управление данными и минимизирует избыточность информации.

Архитектура информационной системы построена на трёхуровневой модели. Уровень представления (презентационный слой) реализован на базе ASP.NET Core Blazor Server: пользовательский веб-интерфейс — страницы заявок, клиентов, товаров, запчастей, продаж, сотрудников, отчётов, администрирования, форма быстрой заявки, вход и регистрация — выполняется на сервере с интерактивным обновлением интерфейса через SignalR; отображение данных и ввод пользователя осуществляются через Razor-компоненты и привязку к данным. Уровень бизнес-логики представлен контроллерами REST API (заявки, клиенты, товары, запчасти, сотрудники, продажи, аутентификация), сервисами приложения (формирование PDF по заявке, отправка почты, при необходимости расчёт стоимости и проверка остатков) и правилами авторизации по ролям; контроллеры и страницы Blazor обращаются к данным через единый слой доступа. Уровень доступа к данным (слой доступа к данным, Data Access Layer) отвечает за взаимодействие с реляционной базой данных и реализован средствами Entity Framework Core: контекст данных ApplicationDbContext наследует IdentityDbContext и определяет наборы сущностей (DbSet) для заявок, клиентов, товаров, запчастей, поставщиков, сотрудников, должностей, видов работ, работ по заявкам, продаж и позиций продаж, а также хранит конфигурацию связей и правил удаления в методе OnModelCreating. Обращение к базе выполняется через типизированные запросы (LINQ к сущностям), сохранение и обновление — через методы Add, Update, Remove и SaveChanges. В качестве СУБД используется PostgreSQL (драйвер Npgsql); при недоступности PostgreSQL предусмотрен резервный вариант с SQLite для упрощённого развёртывания и разработки. Таким образом, уровень доступа к данным обеспечивает абстракцию над реляционной базой и единообразную работу с данными во всём приложении.
