# Логическая модель БД, полученная из физической модели

Ниже представлена логическая модель базы данных информационной системы «ВузяПринт», построенная по физической модели (схема `public`, 12 таблиц). Сущности и атрибуты описаны в терминах предметной области без привязки к типам СУБД.

---

## Соответствие: физическая таблица → логическая сущность

| Физическая таблица (public.*) | Логическая сущность |
|-------------------------------|---------------------|
| employees                     | Сотрудники          |
| positions                     | Должности           |
| clients                       | Клиенты             |
| suppliers                     | Поставщики          |
| orders                        | Заявки на ремонт    |
| parts                         | Запчасти            |
| worktypes                     | Виды работ          |
| products                      | Товары              |
| sales                         | Продажи             |
| orderparts                    | Позиции заявки по запчастям |
| works                         | Работы по заявке    |
| saleitems                     | Позиции продаж      |

---

## 2. Связи между сущностями (логическая модель)

| Родитель (логическая сущность) | Связь | Дочерняя сущность | Внешний ключ в физической модели |
|---------------------------------|-------|-------------------|-----------------------------------|
| Клиенты                        | 1 → * | Заявки на ремонт  | orders.clientid → clients.id      |
| Сотрудники                     | 1 → * | Заявки на ремонт  | orders.employeeid → employees.id  |
| Заявки на ремонт               | 1 → * | Позиции заявки по запчастям | orderparts.orderid → orders.id |
| Запчасти                       | 1 → * | Позиции заявки по запчастям | orderparts.partid → parts.id   |
| Заявки на ремонт               | 1 → * | Работы по заявке  | works.orderid → orders.id         |
| Виды работ                     | 1 → * | Работы по заявке  | works.worktypeid → worktypes.id   |
| Клиенты                        | 1 → * | Продажи           | sales.clientid → clients.id       |
| Продажи                        | 1 → * | Позиции продаж    | saleitems.saleid → sales.id       |
| Товары                         | 1 → * | Позиции продаж    | saleitems.productid → products.id |
| Поставщики                     | 1 → * | Товары            | products.supplierid → suppliers.id|
| Поставщики                     | 1 → * | Запчасти          | parts.supplierid → suppliers.id   |
| Должности                      | 1 → * | Сотрудники        | employees.positionid → positions.id |

**Обозначения:** 1 → * — связь «один ко многим»; FK — внешний ключ.

Логическая модель соответствует третьей нормальной форме (3НФ); в физической реализации связи обеспечиваются внешними ключами и при необходимости правилами каскадного удаления и ограничениями.
