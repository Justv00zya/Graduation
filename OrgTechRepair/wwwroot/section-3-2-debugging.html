3. Технологическая часть (продолжение)

3.2. Отладка программы

Отладка информационной системы «ВузяПринт» (веб-приложение OrgTechRepair на платформе ASP.NET Core, клиентская часть — Blazor Server, СУБД — PostgreSQL) выполнялась в среде Microsoft Visual Studio / JetBrains Rider с использованием встроенного отладчика для управляемого кода C#. Устанавливались точки останова в контроллерах API, в сервисах, в методах жизненного цикла Razor-компонентов и в обработчиках событий интерфейса; выполнялся пошаговый проход (Step Over, Step Into), просмотр значений локальных переменных, полей сущностей и результатов асинхронных вызовов к базе данных через Entity Framework Core.

Для проверки обмена данными между браузером и сервером использовались средства инструментов разработчика веб-обозревателя (вкладки «Сеть», «Консоль»): анализировались HTTP-запросы к конечным точкам /api/…, коды ответов, заголовок Authorization с JSON Web Token при вызовах из мобильного клиента, а также ответы Blazor Server по WebSocket при работе интерактивных страниц. При отладке сценариев входа и регистрации дополнительно проверялись перенаправления форм на контроллер AccountController и выдача cookie-аутентификации.

Аутентификация пользователей в веб-интерфейсе реализована на базе ASP.NET Core Identity (логин или e-mail и пароль); для API мобильного приложения используется схема JWT Bearer. При отладке проверялась корректность назначения ролей, доступ к защищённым маршрутам по атрибуту [Authorize(Roles = …)] и сценарий отказа в доступе с переходом на страницу /AccessDenied. Специфика Blazor Server (долгоживущая сигнальная сессия с сервером) учитывалась при анализе состояния аутентификации и при повторном входе пользователя.

В приложении используется стандартный механизм журналирования ILogger: уровни Information, Warning и Error позволяют отслеживать ошибки бизнес-логики, сбои при обращении к PostgreSQL и исключения при генерации PDF-заявок. Для администратора предусмотрен просмотр накопленных записей лога через интерфейс раздела «Администрирование» и выгрузка в CSV; при разработке также применялся вывод в окно отладки среды.
