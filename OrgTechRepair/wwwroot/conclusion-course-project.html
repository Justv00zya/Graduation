Заключение

В ходе выполнения курсового проекта была поставлена задача разработки информационной системы «ВузяПринт» (веб-приложение OrgTechRepair) для автоматизации учёта заявок на ремонт оргтехники, ведения справочников клиентов, сотрудников, поставщиков, товаров и запчастей, а также для сопровождения процессов оформления заказов, продаж и обращений на поставку комплектующих. Требовалось обеспечить разграничение доступа по ролям пользователей, устойчивое хранение данных в реляционной СУБД и возможность работы с системой не только из браузера, но и с мобильного клиента.

Поставленная цель достигнута. Реализовано серверное приложение на платформе ASP.NET Core с клиентской частью на Blazor Server, REST API для внешних клиентов и подключением к PostgreSQL через Entity Framework Core. Данные о заявках, клиентах, персонале и номенклатуре структурированы в виде связанных таблиц; для веб-пользователей применена аутентификация ASP.NET Core Identity, для мобильного приложения — выдача и проверка JWT с ограничением операций по ролям. Разработано мобильное приложение на Flutter, использующее те же конечные точки API для просмотра и создания заявок в соответствии с правами учётной записи.

В системе предусмотрены формирование и архивирование PDF-заявок, средства журналирования и административный просмотр логов, а также приёмы защитного программирования (проверки целостности данных, обработка исключений при вспомогательных операциях с файловой системой и генерацией документов), что снижает риск некорректного завершения сценариев при локальных сбоях. Проведена отладка и тестирование основных пользовательских сценариев в среде разработки с использованием отладчика и инструментов браузера.

Практическая значимость работы заключается в том, что разработанный комплекс может быть использован в учебном центре или подразделении, обслуживающем оргтехнику, как прототип полноценной учётной системы: он снижает долю ручного ввода и дублирования информации, ускоряет доступ к статусу заявок и справочникам и создаёт основу для дальнейшего развития (расширение отчётности, интеграция с внешними сервисами, доработка мобильного функционала под конкретный регламент организации).
