Таблица — описание всех файлов веб-проекта OrgTechRepair

Строки 1–100: проект OrgTechRepair/OrgTechRepair (исходный код сайта). Строки 101–106: корень репозитория (SQL и файл решения .sln). Не включены bin/, obj/, каталоги IDE и пакеты NuGet. Мобильный клиент OrgTechRepairMobile — отдельный репозиторий/папка; по запросу можно оформить аналогичную таблицу для Dart/Flutter.

№ Путь к файлу (от папки проекта OrgTechRepair) Назначение
1OrgTechRepair.csprojФайл проекта MSBuild: целевая платформа .NET, ссылки на пакеты NuGet (EF Core, Identity, Blazor, QuestPDF и др.), включение Razor/CSS.
2Program.csТочка входа: построение конвейера HTTP, подключение Blazor Server, PostgreSQL/SQLite, Identity, JWT, CORS, регистрация сервисов и контроллеров, инициализация БД и сидирование.
3appsettings.jsonОсновная конфигурация: строки подключения к БД, JWT, путь архива PDF-заявок, логирование, ограничение хостов.
4appsettings.Development.jsonПереопределения для среды Development (например, каталог заявок, строка подключения при локальной отладке).
5Properties/launchSettings.jsonПрофили запуска Kestrel/IIS Express: URL, переменная среды ASPNETCORE_ENVIRONMENT.
6Data/ApplicationDbContext.csКонтекст Entity Framework: DbSet сущностей (заказы, клиенты, сотрудники, товары, запчасти, продажи, заявки на склад и т.д.), связи и ограничения.
7Data/SeedData.csНачальное заполнение БД: роли Identity, демо-пользователи, справочники, тестовые заявки и номенклатура для демонстрации.
8Data/PostgreSqlDbBootstrap.csВспомогательная логика при старте для PostgreSQL (создание/проверка структуры при необходимости).
9Data/ApplyClientColumnsMigration.csПрограммное применение изменений схемы: колонки клиента при отсутствии полноценной миграции EF.
10Data/ApplyDateColumnsMigration.csПрограммное добавление/обновление колонок дат в таблицах по необходимости.
11Data/ApplyPartSupplyRequestsTableMigration.csСоздание или обновление таблицы заявок на выдачу запчастей со склада.
12Data/ApplyProductImageColumnMigration.csДобавление колонки URL изображения товара в таблицу продуктов.
13Models/Client.csСущность клиента сервиса: ФИО/организация, контакты, адрес, связь с пользователем Identity.
14Models/Employee.csСущность сотрудника: табельный номер, ФИО, должность, даты, реквизиты.
15Models/Order.csСущность заявки на ремонт: номер, клиент, модель техники, жалоба, статус, стоимость, исполнитель.
16Models/OrderPart.csСвязь заявки с использованными запчастями (учёт деталей по заказу).
17Models/Part.csСущность запчасти на складе сервиса: код, наименование, поставщик, цена, остаток.
18Models/PartSupplyRequest.csЗаявка инженера на выдачу запчасти со склада: статус, связь с заявкой на ремонт, комментарии.
19Models/Position.csСправочник должностей сотрудников и оклад.
20Models/Product.csСущность товара (расходники): код, название, модель, поставщик, цена, количество, URL изображения.
21Models/Sale.csСущность продажи (документ): дата, клиент, сумма, позиции.
22Models/SaleItem.csСтрока продажи: товар, количество, цена.
23Models/Supplier.csСущность поставщика: наименование, адрес, телефон, ИНН.
24Models/Work.csСвязь выполненных работ с заявкой (вид работ, стоимость в контексте заказа).
25Models/WorkType.csСправочник видов работ и базовая цена.
26Models/DTOs/ClientDto.csОбъект передачи данных клиента для API и UI.
27Models/DTOs/EmployeeDto.csDTO сотрудника для API (без лишних навигационных полей).
28Models/DTOs/OrderDto.csDTO заявки на ремонт для списков и деталей в API.
29Models/DTOs/PartSupplyRequestDtos.csDTO создания и отображения заявок на выдачу запчастей.
30Models/DTOs/ProductDto.csDTO товара для REST и форм редактирования.
31Models/DTOs/QuickRequestDto.csПоля быстрой заявки с главной страницы (имя, телефон, модель, описание проблемы).
32Models/DTOs/SupplierDto.csDTO поставщика для API.

35Services/DevelopmentEmailSender.csЗаглушка отправки e-mail в разработке (логирование вместо реальной отправки).

48Controllers/HealthController.csПроверка доступности API (health/ping) для мониторинга.
49Controllers/OrdersController.csREST заявок на ремонт, быстрая анонимная заявка, генерация PDF при сохранении.
50Controllers/PartsController.csREST запчастей: список, получение по id, создание, обновление, удаление.
51Controllers/PartSupplyRequestsController.csREST заявок инженера на запчасти: создание, список, одобрение/отклонение кладовщиком.
52Controllers/ProductsController.csREST товаров: список, CRUD, загрузка изображения карточки товара.
53Controllers/ReportsController.csAPI отчётов и агрегированных данных для страницы отчётов.
54Controllers/SalesController.csREST продаж: список, детали, создание продаж с позициями.
55Controllers/SuppliersController.csREST поставщиков: CRUD для справочника.
