Логическая модель базы данных (ER-модель)

Информационная система «ВузяПринт». Выделите нужный фрагмент или таблицу, Ctrl+C, в Word — Ctrl+V. Либо откройте файл в Word (Файл → Открыть) и сохраните как .docx.

1. Сущности и атрибуты

Orders (Заявки)
  • Id (PK), OrderNumber, ClientId (FK → Clients), EquipmentModel, ConditionDescription, ComplaintDescription, EmployeeId (FK → Employees), Cost, OrderDate, CompletionDate, Status
OrderParts (Позиции заявки по запчастям)
  • Id (PK), OrderId (FK → Orders), PartId (FK → Parts), Quantity
Works (Работы по заявке)
  • Id (PK), OrderId (FK → Orders), WorkTypeId (FK → WorkTypes), SequenceNumber
WorkTypes (Виды работ)
  • Id (PK), Name, Price
Clients (Клиенты)
  • Id (PK), FullName, Address, Phone
Sales (Продажи)
  • Id (PK), SaleNumber, ClientId (FK → Clients), SaleDate, TotalAmount
SaleItems (Позиции продаж)
  • Id (PK), SaleId (FK → Sales), ProductId (FK → Products), Quantity, UnitPrice, TotalPrice
Products (Товары)
  • Id (PK), Code, SupplierId (FK → Suppliers), Name, Model, Price, Quantity
Parts (Запчасти)
  • Id (PK), Code, SupplierId (FK → Suppliers), Name, Price, Quantity
Suppliers (Поставщики)
  • Id (PK), Name, Address, INN, AccountNumber, Phone
Employees (Сотрудники)
  • Id (PK), TabNumber, PositionId (FK → Positions), FirstName, LastName, MiddleName, DateOfBirth, INN, Address, HireDate
Positions (Должности)
  • Id (PK), Name, Salary
AspNetUsers, AspNetRoles, AspNetUserRoles
  • Таблицы ASP.NET Core Identity: пользователи, роли, связь «пользователь — роль» (многие ко многим).

Родитель Связь Дочерняя сущность Внешний ключ Поведение при удалении

WorkTypes1 → *WorksWork.WorkTypeIdRestrict

Пояснения: 1 → * — «один ко многим»; Restrict — удаление родителя запрещено при наличии связанных записей; Cascade — при удалении родителя удаляются дочерние записи; SetNull — внешний ключ в дочерней записи обнуляется.
