№ Ситуация Аномалия Реакция Примечание
1 Авторизация (вход) Попытка доступа к защищённым разделам без входа в систему Программа перенаправляет пользователя на страницу входа Доступ к заявкам, клиентам, товарам и другим разделам возможен только после входа под учётной записью
2 Авторизация (вход) Ввод неверного логина или пароля Программа отклоняет вход и выводит сообщение об ошибке аутентификации Пароль хранится в виде хеша; проверка выполняется на сервере
3 Авторизация (роли) Попытка доступа к разделу, к которому у роли пользователя нет прав (например, «Администрирование» для менеджера) Программа перенаправляет на страницу «Доступ запрещён» или возвращает HTTP 403 при обращении к API Проверка прав выполняется на сервере при каждом запросе
4 Регистрация пользователя Имя пользователя или e-mail уже существуют в системе Программа не создаёт учётную запись и выводит сообщение об ошибке Уникальность логина и при необходимости e-mail проверяется при сохранении
5 Регистрация пользователя Пароль не соответствует политике (например, менее 6 символов) Программа не создаёт учётную запись и выводит сообщение о требованиях к паролю Политика задаётся в настройках ASP.NET Core Identity
6 Заявка на ремонт Создание или сохранение заявки с несуществующим идентификатором клиента Программа не сохраняет заявку и выводит сообщение об ошибке валидации Внешний ключ ClientId ссылается на справочник клиентов; целостность обеспечивается БД и EF Core
7 Заявка на ремонт Удаление клиента, у которого есть заявки или продажи Программа запрещает удаление (ограничение по внешнему ключу) и выводит сообщение об ошибке Настроено поведение OnDelete(DeleteBehavior.Restrict) для связи заявка — клиент
8 Позиции заявки (запчасти) Добавление к заявке запчасти с количеством, превышающим остаток на складе При реализации проверки остатков программа не сохраняет позицию или выводит предупреждение (в соответствии с логикой приложения) Контроль остатков может выполняться в бизнес-логике при сохранении заявки
9 Товары и запчасти Удаление поставщика, у которого есть привязанные товары или запчасти Программа запрещает удаление поставщика и выводит сообщение об ошибке Используется OnDelete(DeleteBehavior.Restrict) для связей товар — поставщик, запчасть — поставщик
10 Продажа Оформление позиции продажи с количеством товара, превышающим остаток на складе При реализации проверки остатков программа не сохраняет продажу или выводит сообщение об ошибке Остаток товара должен быть неотрицательным; проверка при сохранении позиций продажи
11 Справочники Ввод отрицательной цены или отрицательного количества для товара, запчасти, вида работ или должности Программа не сохраняет запись и выводит сообщение об ошибке валидации Проверка на стороне клиента (формы) и на сервере (атрибуты, проверка в коде)
12 API (REST) Обращение к защищённому методу API без JWT-токена или с недействительным/истёкшим токеном Сервер возвращает ответ с кодом 401 Unauthorized; тело ответа содержит сообщение об ошибке Проверка токена выполняется middleware аутентификации JWT для всех запросов к /api/*
