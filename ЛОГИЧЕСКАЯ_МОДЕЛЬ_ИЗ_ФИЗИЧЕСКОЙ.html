Логическая модель БД, полученная из физической модели

Информационная система «ВузяПринт». Логическая модель построена по физической (схема public, 12 таблиц). Для копирования в Word: выделите таблицу или фрагмент → Ctrl+C, в Word → Ctrl+V.

Соответствие: физическая таблица → логическая сущность

2. Связи между сущностями (логическая модель)

Родитель (логическая сущность) Связь Дочерняя сущность Внешний ключ в физической модели
Клиенты1 → *Заявки на ремонтorders.clientid → clients.id
Сотрудники1 → *Заявки на ремонтorders.employeeid → employees.id
Заявки на ремонт1 → *Позиции заявки по запчастямorderparts.orderid → orders.id
Запчасти1 → *Позиции заявки по запчастямorderparts.partid → parts.id
Заявки на ремонт1 → *Работы по заявкеworks.orderid → orders.id
Виды работ1 → *Работы по заявкеworks.worktypeid → worktypes.id
Клиенты1 → *Продажиsales.clientid → clients.id
Продажи1 → *Позиции продажsaleitems.saleid → sales.id
Товары1 → *Позиции продажsaleitems.productid → products.id
Поставщики1 → *Товарыproducts.supplierid → suppliers.id
Поставщики1 → *Запчастиparts.supplierid → suppliers.id
Должности1 → *Сотрудникиemployees.positionid → positions.id

Обозначения: 1 → * — связь «один ко многим»; FK — внешний ключ. Логическая модель соответствует третьей нормальной форме (3НФ).
