Таблица 6 — Описание классов

Выделите таблицу мышью, нажмите Ctrl+C, в документе Word — Ctrl+V. Либо откройте файл в Word (Файл → Открыть) и сохраните как .docx.

№ Наименование класса Назначение класса
1AuthControllerКонтроллер авторизации: выдача JWT по логину и паролю (POST /api/auth/login), регистрация пользователя (POST /api/auth/register).
2AccountControllerWeb-контроллер учётной записи: страницы входа, регистрации, восстановления и сброса пароля, управление cookie-аутентификацией.
3OrdersControllerREST API заявок на ремонт: получение списка и одной заявки, создание, обновление, удаление (CRUD).
4ClientsControllerREST API клиентов: получение списка и одного клиента, создание, обновление, удаление.
5ProductsControllerREST API товаров: получение списка и одного товара, создание, обновление, удаление.
6PartsControllerREST API запчастей: получение списка и одной запчасти, создание, обновление, удаление.
7EmployeesControllerREST API сотрудников: получение списка и одного сотрудника (просмотр).
8SalesControllerREST API продаж: получение списка и одной продажи с составом позиций.
9ApplicationDbContextКонтекст данных Entity Framework Core: наборы сущностей (DbSet), конфигурация связей и правил удаления (OnModelCreating), наследует IdentityDbContext для пользователей и ролей.
10SeedDataСтатический класс начального заполнения БД: создание ролей, учётной записи администратора, справочников (должности, виды работ, при необходимости демо-данные).
11RevalidatingIdentityAuthenticationStateProviderПровайдер состояния аутентификации для Blazor Server: проверка актуальности пользователя при взаимодействии с сервером.
12IOrderPdfServiceИнтерфейс сервиса формирования PDF-документа по заявке на ремонт.
13OrderPdfServiceРеализация сервиса генерации PDF по заявке (библиотека QuestPDF).
14IEmailSenderИнтерфейс отправки электронной почты (восстановление пароля, уведомления).
15DevelopmentEmailSenderРеализация отправки почты для разработки: вывод содержимого писем в лог.
16OrderМодель заявки на ремонт: номер, клиент, модель оборудования, описание неисправности, исполнитель, стоимость, даты, статус, связи с позициями запчастей и работами.
17OrderPartМодель позиции заявки по запчасти: заявка, запчасть, количество.
18WorkМодель выполненной работы по заявке: заявка, вид работ, порядковый номер.
19WorkTypeМодель вида работ: наименование, цена.
20ClientМодель клиента: ФИО или наименование, адрес, телефон.
21ProductМодель товара: код, поставщик, наименование, модель, цена, количество на складе.
22PartМодель запчасти: код, поставщик, наименование, цена, количество на складе.
23SupplierМодель поставщика: наименование, адрес, ИНН, расчётный счёт, телефон.
24EmployeeМодель сотрудника: табельный номер, должность, ФИО, дата рождения, ИНН, адрес, дата приёма.
25PositionМодель должности: наименование, оклад.
26SaleМодель продажи: номер, клиент, дата, итоговая сумма, состав позиций.
27SaleItemМодель позиции продажи: продажа, товар, количество, цена за единицу, сумма по позиции.
28OrderDtoDTO заявки для API: передача данных заявки с связанными сущностями (клиент, исполнитель и т.д.).
29CreateOrderDtoDTO создания заявки: поля, необходимые для POST-запроса.
30UpdateOrderDtoDTO обновления заявки: поля для PUT-запроса.
31ClientDtoDTO клиента для API.
32CreateClientDtoDTO создания клиента.
33UpdateClientDtoDTO обновления клиента.
34ProductDtoDTO товара для API.
35CreateProductDtoDTO создания товара.
36UpdateProductDtoDTO обновления товара.
37EmployeeDtoDTO сотрудника для API.
38LoginRequestМодель запроса входа в API: имя пользователя (логин) и пароль для POST /api/auth/login.
39LoginResponseМодель ответа API при успешном входе: JWT-токен и при необходимости срок действия.
40RegisterRequestМодель запроса регистрации в API: логин, пароль, электронная почта для POST /api/auth/register.
