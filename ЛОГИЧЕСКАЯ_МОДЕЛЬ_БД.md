# Логическая модель базы данных (ER-диаграмма)

Информационная система «ВузяПринт» использует реляционную базу данных. Ниже приведены сущности (таблицы), их атрибуты и связи.

---

## 1. Сущности и атрибуты

### Пользователи и роли (ASP.NET Core Identity)

- **AspNetUsers** (Пользователи)  
  ID (PK), UserName, NormalizedUserName, Email, NormalizedEmail, EmailConfirmed, PasswordHash, SecurityStamp, ConcurrencyStamp, PhoneNumber, PhoneNumberConfirmed, TwoFactorEnabled, LockoutEnd, LockoutEnabled, AccessFailedCount.

- **AspNetRoles** (Роли)  
  Id (PK), Name, NormalizedName, ConcurrencyStamp.

- **AspNetUserRoles** (Пользователь_Роли) — связь «многие ко многим»  
  UserId (FK → AspNetUsers), RoleId (FK → AspNetRoles).

*(При необходимости в документе можно добавить остальные таблицы Identity: AspNetUserClaims, AspNetUserLogins, AspNetUserTokens, AspNetRoleClaims.)*

---

### Заявки на ремонт

- **Orders** (Заявки)  
  Id (PK), OrderNumber (номер заявки), ClientId (FK → Clients), EquipmentModel (модель оборудования), ConditionDescription (описание состояния), ComplaintDescription (описание неисправности), EmployeeId (FK → Employees, nullable), Cost (стоимость), OrderDate (дата приёма), CompletionDate (дата выполнения), Status (статус: Принят, В работе, Выполнен, Отменён).

- **OrderParts** (Позиции_заявки_по_запчастям)  
  Id (PK), OrderId (FK → Orders), PartId (FK → Parts), Quantity (количество).

- **Works** (Работы_по_заявке)  
  Id (PK), OrderId (FK → Orders), WorkTypeId (FK → WorkTypes), SequenceNumber (порядковый номер).

- **WorkTypes** (Виды_работ)  
  Id (PK), Name (наименование), Price (цена).

---

### Клиенты и продажи

- **Clients** (Клиенты)  
  Id (PK), FullName (ФИО или наименование), Address (адрес), Phone (телефон).

- **Sales** (Продажи)  
  Id (PK), SaleNumber (номер продажи), ClientId (FK → Clients, nullable), SaleDate (дата продажи), TotalAmount (итоговая сумма).

- **SaleItems** (Позиции_продаж)  
  Id (PK), SaleId (FK → Sales), ProductId (FK → Products), Quantity (количество), UnitPrice (цена за единицу), TotalPrice (сумма по позиции).

---

### Товары и запчасти

- **Products** (Товары)  
  Id (PK), Code (код товара), SupplierId (FK → Suppliers), Name (наименование), Model (модель), Price (цена), Quantity (количество на складе).

- **Parts** (Запчасти)  
  Id (PK), Code (код запчасти), SupplierId (FK → Suppliers), Name (наименование), Price (цена), Quantity (количество на складе).

- **Suppliers** (Поставщики)  
  Id (PK), Name (наименование), Address (адрес), INN (ИНН), AccountNumber (расчётный счёт), Phone (телефон).

---

### Сотрудники

- **Employees** (Сотрудники)  
  Id (PK), TabNumber (табельный номер), PositionId (FK → Positions), FirstName (имя), LastName (фамилия), MiddleName (отчество), DateOfBirth (дата рождения), INN (ИНН), Address (адрес), HireDate (дата приёма на работу).

- **Positions** (Должности)  
  Id (PK), Name (наименование должности), Salary (оклад).

---

## 2. Связи между сущностями

| Родительская сущность | Связь | Дочерняя сущность | Внешний ключ | Удаление |
|-----------------------|-------|-------------------|--------------|----------|
| Clients               | 1 → * | Orders            | Order.ClientId | Restrict |
| Employees             | 1 → * | Orders            | Order.EmployeeId | SetNull |
| Orders                | 1 → * | OrderParts        | OrderPart.OrderId | Cascade |
| Parts                 | 1 → * | OrderParts        | OrderPart.PartId | Restrict |
| Orders                | 1 → * | Works             | Work.OrderId | Cascade |
| WorkTypes             | 1 → * | Works             | Work.WorkTypeId | Restrict |
| Clients               | 1 → * | Sales             | Sale.ClientId | SetNull |
| Sales                 | 1 → * | SaleItems         | SaleItem.SaleId | Cascade |
| Products              | 1 → * | SaleItems         | SaleItem.ProductId | Restrict |
| Suppliers             | 1 → * | Products          | Product.SupplierId | Restrict |
| Suppliers             | 1 → * | Parts             | Part.SupplierId | Restrict |
| Positions             | 1 → * | Employees         | Employee.PositionId | Restrict |
| AspNetUsers           | 1 → * | AspNetUserRoles   | UserId | — |
| AspNetRoles           | 1 → * | AspNetUserRoles   | RoleId | — |

**Пояснения:**  
- **1 → *** — «один ко многим»: одной записи родителя соответствует несколько записей дочерней таблицы.  
- **Restrict** — при удалении родителя удаление запрещено, если есть связанные записи.  
- **Cascade** — при удалении родителя удаляются связанные дочерние записи.  
- **SetNull** — при удалении родителя внешний ключ в дочерней записи устанавливается в NULL (если поле допускает NULL).

---

## 3. Краткая схема связей (текстом)

- **Заявка (Order)** связана с одним **клиентом (Client)** и опционально с одним **исполнителем (Employee)**; содержит несколько **позиций по запчастям (OrderPart)** и несколько **работ (Work)**. Каждая позиция ссылается на одну **запчасть (Part)**; каждая работа — на один **вид работ (WorkType)**.
- **Продажа (Sale)** связана опционально с одним **клиентом (Client)** и содержит несколько **позиций продаж (SaleItem)**. Каждая позиция ссылается на один **товар (Product)**.
- **Товар (Product)** и **запчасть (Part)** связаны с одним **поставщиком (Supplier)**.
- **Сотрудник (Employee)** связан с одной **должностью (Position)**.
- **Пользователи (AspNetUsers)** и **роли (AspNetRoles)** связаны через **AspNetUserRoles** (многие ко многим).

Логическая модель приведена к третьей нормальной форме (3НФ); связи реализованы через внешние ключи в реляционной СУБД (PostgreSQL / SQLite).
