# Регистрация и восстановление пароля

Документация по функционалу регистрации пользователей и восстановления пароля в приложении OrgTechRepair.

## Реализованный функционал

### 1. Регистрация пользователей

**Страница:** `/Register`

**Описание:**
- Позволяет новым пользователям создать аккаунт в системе
- Требует ввода логина, email, пароля и подтверждения пароля
- Валидация пароля соответствует требованиям Identity:
  - Минимум 6 символов
  - Должна быть заглавная буква
  - Должна быть строчная буква
  - Должна быть цифра
- После успешной регистрации пользователь может войти в систему

**Особенности:**
- Email автоматически подтверждается при регистрации (для упрощения)
- В продакшене рекомендуется добавить подтверждение email через письмо

### 2. Восстановление пароля

**Страница:** `/ForgotPassword`

**Описание:**
- Позволяет пользователям запросить восстановление пароля
- Требует ввода email адреса
- Генерирует токен восстановления пароля
- В режиме разработки показывает ссылку для сброса пароля на странице
- В продакшене ссылка отправляется на email

**Процесс:**
1. Пользователь вводит email
2. Система генерирует токен восстановления
3. Создается ссылка вида: `/ResetPassword?email={email}&token={token}`
4. В режиме разработки ссылка отображается на странице
5. В продакшене ссылка отправляется на email через `IEmailSender`

### 3. Сброс пароля

**Страница:** `/ResetPassword`

**Описание:**
- Позволяет пользователю установить новый пароль по токену
- Токен передается через URL параметры
- Требует ввода нового пароля и подтверждения
- После успешного сброса пароля пользователь может войти с новым паролем

**Безопасность:**
- Токены имеют ограниченный срок действия (настраивается в Identity)
- Токены одноразовые (после использования становятся недействительными)
- Для безопасности не раскрывается информация о существовании пользователя

## Навигация

### Для неавторизованных пользователей:
- **Вход** (`/Login`) - страница входа в систему
- **Регистрация** (`/Register`) - страница регистрации
- Ссылка "Забыли пароль?" на странице входа

### Для авторизованных пользователей:
- Все функции доступны через главное меню

## Email сервис

### Реализация для разработки

В режиме разработки используется `DevelopmentEmailSender`, который:
- Выводит ссылки восстановления пароля в логи консоли
- Показывает ссылку на странице для удобства тестирования
- Не требует настройки SMTP сервера

### Реализация для продакшена

Для продакшена необходимо:

1. **Создать реализацию IEmailSender** с реальной отправкой email:
   ```csharp
   public class SmtpEmailSender : IEmailSender
   {
       // Реализация через SMTP, SendGrid, Mailgun и т.д.
   }
   ```

2. **Зарегистрировать в Program.cs:**
   ```csharp
   builder.Services.AddScoped<IEmailSender, SmtpEmailSender>();
   ```

3. **Настроить параметры SMTP в appsettings.json:**
   ```json
   {
     "EmailSettings": {
       "SmtpServer": "smtp.example.com",
       "SmtpPort": 587,
       "Username": "your-email@example.com",
       "Password": "your-password",
       "FromEmail": "noreply@printservice.ru",
       "FromName": "ВузяПринт"
     }
   }
   ```

## База данных

### Таблицы Identity

Все необходимые таблицы для регистрации и восстановления пароля уже включены в структуру БД:

- **AspNetUsers** - хранит информацию о пользователях
- **AspNetUserTokens** - хранит токены восстановления пароля и другие токены
- **AspNetRoles** - роли пользователей
- **AspNetUserRoles** - связь пользователей и ролей

### Токены восстановления пароля

Токены хранятся в таблице `AspNetUserTokens` с:
- `LoginProvider` = "Default" (или имя провайдера)
- `Name` = "PasswordResetToken"
- `Value` = зашифрованный токен
- `UserId` = ID пользователя

Токены автоматически управляются через `UserManager`:
- `GeneratePasswordResetTokenAsync()` - генерация токена
- `ResetPasswordAsync()` - сброс пароля по токену

## Безопасность

### Требования к паролю

Настроены в `Program.cs`:
```csharp
options.Password.RequireDigit = true;
options.Password.RequireLowercase = true;
options.Password.RequireUppercase = true;
options.Password.RequireNonAlphanumeric = false;
options.Password.RequiredLength = 6;
```

### Защита от атак

1. **Токены имеют срок действия** - настраивается через `TokenOptions`
2. **Одноразовые токены** - после использования становятся недействительными
3. **Не раскрывается информация** - даже если email не найден, показывается успех
4. **CSRF защита** - используется `FormName` атрибут в Blazor формах

## Использование

### Регистрация нового пользователя

1. Перейдите на `/Register`
2. Заполните форму:
   - Логин (минимум 3 символа)
   - Email (должен быть валидным)
   - Пароль (минимум 6 символов, с заглавной, строчной буквой и цифрой)
   - Подтверждение пароля
3. Нажмите "Зарегистрироваться"
4. После успешной регистрации перейдите на страницу входа

### Восстановление пароля

1. На странице входа нажмите "Забыли пароль?"
2. Введите email адрес, который использовался при регистрации
3. В режиме разработки:
   - Ссылка для сброса пароля будет показана на странице
   - Также можно скопировать ссылку из логов консоли
4. В продакшене:
   - Ссылка будет отправлена на email
5. Перейдите по ссылке для сброса пароля
6. Введите новый пароль и подтверждение
7. После успешного сброса войдите с новым паролем

## API

### Регистрация через API

Регистрация через API доступна только для администраторов:

```http
POST /api/auth/register
Authorization: Bearer {token}
Content-Type: application/json

{
  "username": "newuser",
  "email": "user@example.com",
  "password": "Password123",
  "role": "Manager"
}
```

### Восстановление пароля через API

Восстановление пароля через API не реализовано, используйте веб-интерфейс.

## Настройка для продакшена

### 1. Настройка email сервиса

Создайте реализацию `IEmailSender` с реальной отправкой email.

### 2. Настройка токенов

В `Program.cs` можно настроить срок действия токенов:

```csharp
builder.Services.Configure<DataProtectionTokenProviderOptions>(options =>
{
    options.TokenLifespan = TimeSpan.FromHours(24); // Токен действителен 24 часа
});
```

### 3. Подтверждение email

Рекомендуется добавить подтверждение email при регистрации:
- Генерировать токен подтверждения
- Отправлять на email
- Требовать подтверждения перед первым входом

### 4. Логирование

Все операции регистрации и восстановления пароля логируются для аудита.

## Тестирование

### Тест регистрации

1. Откройте `/Register`
2. Заполните форму валидными данными
3. Проверьте успешную регистрацию
4. Войдите с новыми учетными данными

### Тест восстановления пароля

1. Откройте `/ForgotPassword`
2. Введите email существующего пользователя
3. Скопируйте ссылку из страницы или логов
4. Откройте ссылку в браузере
5. Введите новый пароль
6. Войдите с новым паролем

## Известные ограничения

1. **Email подтверждение** - в текущей реализации email автоматически подтверждается при регистрации
2. **Режим разработки** - ссылки восстановления показываются на странице (небезопасно для продакшена)
3. **API восстановления** - восстановление пароля доступно только через веб-интерфейс
